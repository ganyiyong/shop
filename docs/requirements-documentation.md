# 电商库存系统 - 需求文档

## 1. 文档目的

本文档根据当前项目源码反向整理系统需求，用于后续维护、迭代和验收。当前系统定位为个人或小团队电商库存经营管理工具。

## 2. 产品目标

系统目标是帮助经营者完成以下工作：

- 维护商品基础档案。
- 按批次管理进货库存。
- 快速录入销售并自动扣减库存。
- 统计销售额、利润、库存金额和库存数量。
- 记录额外成本并计算净利润。
- 按月维护资产、贷款、待确认货款和存款估算。

## 3. 用户角色

当前系统只有一个角色：

| 角色 | 说明 |
| --- | --- |
| 管理员 | 登录系统后可访问全部功能，完成商品、库存、销售、成本和资产管理 |

当前没有多用户、角色权限或操作审计需求。

## 4. 功能需求

### 4.1 登录认证

需求：

- 系统应提供登录页。
- 用户输入正确账号密码后进入首页看板。
- 未登录用户访问业务页面时应跳转到登录页。
- 用户应能退出登录。

验收标准：

- 正确账号可登录。
- 错误账号或密码不能登录。
- 未登录访问 `/dashboard`、`/goods`、`/stock` 等页面会跳转 `/login`。
- 点击退出后 Session 失效。

### 4.2 首页看板

需求：

- 系统应展示今日销售额、今日利润、今日订单数。
- 系统应展示本月收入、本月利润、本月销售额、本月订单数。
- 系统应展示库存总金额和库存数量。
- 系统应展示总销售额、总利润、年度销售额、年度利润。
- 系统应展示月度销售趋势。
- 系统应展示库存金额排名。
- 系统应支持从首页快速录入销售。

验收标准：

- 首页访问地址 `/dashboard` 可打开。
- 指标能根据销售、库存、额外成本数据变化。
- 快速录入销售成功后返回首页，销售流水增加，库存减少。

### 4.3 商品管理

需求：

- 系统应支持新增商品。
- 系统应支持编辑商品名称、类型、厂家、型号、图片地址、关键词、排序、状态和备注。
- 系统应支持按商品和关键词筛选商品列表。
- 系统应支持分页展示商品。
- 系统应支持删除商品。
- 删除商品后，商品不应在默认商品列表中展示。

验收标准：

- 新增商品后列表可见。
- 编辑商品后列表展示更新。
- 删除商品后 `deleted` 标记为 `1`，默认列表不显示。
- 停用商品不出现在销售和库存的可选启用商品列表中。

### 4.4 库存管理

需求：

- 系统应支持按商品录入库存批次。
- 每个库存批次应记录箱数、盒数、当前库存、进货单价、发出成本和销售价格。
- 系统应自动计算入库总金额和当前库存金额。
- 系统应支持编辑库存批次。
- 系统应支持删除库存批次。
- 系统应支持默认查看有效库存，也支持查看全部库存。
- 系统应支持按商品筛选库存。

验收标准：

- 录入库存时，若当前库存为空，则默认等于盒数。
- 入库总金额等于盒数乘进货单价。
- 当前库存金额等于当前库存乘进货单价。
- 有效库存列表只展示 `stock > 0` 的记录。

### 4.5 库存统计

需求：

- 系统应按库存批次展示仍有库存的数据。
- 系统应支持按商品过滤库存统计。
- 系统应支持按库存数量或库存金额排序。
- 系统应支持升序和降序排序。
- 系统应展示查询结果的库存数量合计和库存金额合计。

验收标准：

- `/inventory-stats` 展示库存统计页。
- 只统计 `stock > 0` 的库存批次。
- 切换排序字段和排序方向后结果顺序变化。

### 4.6 销售管理

需求：

- 系统应支持录入销售。
- 录入销售时应选择商品，可选择库存批次。
- 录入销售时应选择店铺、商品来源，并填写销售价和备注。
- 系统应根据店铺自动确定平台。
- 系统应根据库存批次确定成本价。
- 系统应自动计算手续费和利润。
- 系统应在销售保存后扣减库存 1 件。
- 系统应支持按日期范围和商品筛选销售流水。
- 系统应支持销售明细和商品统计两种视图。
- 系统应支持删除销售流水。

验收标准：

- 销售保存成功后 `t_shop_sale` 增加记录。
- 若选择库存批次，对应库存减少 1。
- 若未选择库存批次，系统自动扣减该商品最早的可用库存。
- 手续费等于销售价乘 0.006，并四舍五入到两位小数。
- 利润等于销售价减成本价减手续费。
- 删除销售流水后，销售记录消失，但库存不自动恢复。

### 4.7 额外成本管理

需求：

- 系统应支持按月份查看额外成本。
- 系统应支持新增额外成本，记录类目、金额和备注。
- 系统应支持删除额外成本。
- 额外成本应参与首页净利润、本月利润、年度利润计算。

验收标准：

- 新增成本后当月成本列表可见。
- 删除成本后当月成本列表不可见。
- 成本变化会影响首页相关利润指标。

### 4.8 资产管理

需求：

- 系统应支持按月份查看资产快照。
- 系统应支持维护现金存款、进货金额、贷款、资产项目、店铺待确认货款和备注。
- 系统应支持维护资产预估系数。
- 系统应自动计算实际存款、预估存款、今年利润和本月存款。
- 如果本月没有保存进货金额，系统应默认使用当前库存金额。
- 去年存款应优先读取去年 12 月资产快照的实际存款；如果没有记录，则允许手动填写。

验收标准：

- 同一月份重复保存资产时更新原记录。
- 实际存款公式正确。
- 预估存款使用维护的预估系数。
- 维护预估系数后资产页展示公式中的系数变化。

## 5. 非功能需求

### 5.1 易用性

- 页面应支持桌面和移动端基础使用。
- 常用业务应通过侧边栏进入。
- 销售录入应支持弹窗操作，减少页面跳转。

### 5.2 性能

- 常规列表应分页查询。
- 统计查询应以 MySQL 聚合为主。
- 当前使用场景面向小规模经营数据，暂未设计大数据量分库分表。

### 5.3 安全性

- 业务页面必须登录后访问。
- 表单提交应使用后端参数绑定和 SQL 参数占位。
- 生产环境应避免使用硬编码账号密码。

### 5.4 数据可靠性

- 销售保存和库存扣减应在同一事务中完成。
- 系统启动建表不得删除已有数据。
- 建议生产环境定期备份数据库。

### 5.5 兼容性

- 后端运行在 Java 8。
- 数据库连接使用 MySQL Connector/J 8.0.33。
- 页面使用现代浏览器访问。

## 6. 数据需求

### 6.1 商品数据

商品必须至少包含：

- 商品名称
- 类型
- 厂家

系统支持可选字段：

- 型号
- 图片地址
- 关键词
- 排序
- 状态
- 备注

### 6.2 库存数据

库存必须关联商品，并至少包含：

- 盒数
- 进货单价

系统根据录入值计算库存金额。

### 6.3 销售数据

销售必须包含：

- 商品或库存批次
- 店铺
- 销售价
- 商品来源

系统自动补充：

- 平台
- 成本价
- 手续费
- 利润
- 创建时间

### 6.4 资产数据

资产快照以 `yyyy-MM` 月份为唯一标识，保存后用于当月资产统计和后续年度利润对比。

## 7. 业务规则需求

| 编号 | 规则 |
| --- | --- |
| BR-001 | 商品删除为软删除 |
| BR-002 | 库存删除为物理删除 |
| BR-003 | 销售删除为物理删除且不回滚库存 |
| BR-004 | 销售保存扣减库存 1 件 |
| BR-005 | 手续费比例固定为 0.6% |
| BR-006 | 资产预估系数默认值为 0.4 |
| BR-007 | 月度资产快照按月份唯一 |
| BR-008 | 时间范围查询采用左闭右开区间 |

## 8. 后续可选需求

以下能力当前源码未实现，可作为后续迭代方向：

- 多用户和角色权限。
- 操作日志和误删恢复。
- 销售删除自动回滚库存或提供冲销机制。
- 退款、退货和换货流程。
- 批量导入商品、库存和销售。
- 导出 Excel 报表。
- 财务金额统一使用 `BigDecimal`。
- 使用 Spring Security 替代自定义拦截器。
- 配置化手续费比例、店铺与平台映射。
- 库存预警和低库存提醒。
